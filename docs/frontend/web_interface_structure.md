# 前端介面結構與修改規範

為確保開發溝通精確，本檔案定義了 `AdminPanel.vue` (管理端) 與 `App.vue` (玩家端) 的介面區塊名稱。
每次修改程式碼前，必須明確告知使用者正在修改哪一個區塊。

---

# 🛡️ 管理端 (Admin UI)

**檔案路徑**：`client/src/components/AdminPanel.vue`

## 1. 遊戲列表頁 (Dashboard)

對應代碼：`v-if="viewMode === 'dashboard'"`

- **[列表區]**
  - 顯示遊戲代碼、人數、狀態。
  - 特徵：直書按鈕、兩欄資訊佈局。
- **[建立區]**
  - 輸入人數、建立房間。
  - 包含「書本 📖」設定按鈕。
- **[技能設定視窗]** (Modal)
  - 點擊書本後彈出。
  - 包含：全選功能、技能勾選列表。

## 2. 遊戲控制台頁 (Control Panel)

對應代碼：`v-else-if="viewMode === 'control'"`

- **[控制台-頂部資訊]**
  - 顯示回合數、階段、遊戲代碼、返回按鈕 (⬅)。
- **[控制台-流程控制]**
  - 按鈕群組：開始遊戲、進入攻擊/競標階段、強制結束。
- **[控制台-玩家管理]**
  - 玩家卡片列表。
  - 功能：調整 HP、顯示屬性、踢出玩家 (✖)。
- **[控制台-紀錄區]**
  - 最下方的 Log 捲動視窗。
- **[控制台-結算區]**
  - 遊戲結束後的排名顯示。
- **[控制台-彈窗]**
  - 踢人確認、強制結束確認等 Modal。

---

# 🎮 玩家端 (Player UI)

**檔案路徑**：`client/src/App.vue`

## 1. 入口頁面

- **[登入頁]** (`uiState === 'login'/'rejoin'`)
  - 建立角色、代碼重返、管理員登入按鈕。
- **[專屬代碼頁]** (`uiState === 'showCode'`)
  - 顯示玩家專屬代碼。

## 2. 遊戲主畫面 (In Game)

對應代碼：`uiState === 'inGame'`

- **[個人儀表板] (Dashboard)**
  - 固定於頂部。
  - 顯示：屬性(背景色)、HP、等級、持有技能、[進化升級按鈕]。
- **[死亡覆蓋層]**
  - 當 HP <= 0 時覆蓋全螢幕。

### 因應階段變化的底部區域

- **[大廳區] (Waiting)**
  - 顯示遊戲代碼、已加入玩家名單。
- **[討論區] (Discussion)**
  - **[其它玩家列表]**：卡片顯示，包含 下毒/查看/偵查 按鈕。
  - **[可用技能區]**：底部的主動技能按鈕列 (冬眠、瞪人...)。
- **[攻擊區] (Attack)**
  - 類似討論區，卡片按鈕變為「攻擊」。
- **[競標區] (Auction)**
  - **[競標總覽]**：顯示所有技能卡片狀態。
  - **[競標彈窗] (Modal)**：主要的出價介面、HP 柱狀圖、出價 Input。
- **[結算區] (Finished)**
  - 顯示最終排名列表。

## 3. 通用彈窗

- **[規則與確認]**：規則書、偵查確認、冬眠確認、技能目標選擇器。
