# Agent 安全操作協定

## 1. 修改前確認

- **計畫優先**：在進行**架構性重構**或**涉及核心邏輯（參見 01 守則）**的跨檔案修改前，Agent 必須先產出一份 `IMPLEMENTATION_PLAN.md` 供開發者審核。對於**單純的變數更名、樣式調整或 Bug 修復**，可在對話中直接說明並執行。
- **影響評估**：計畫中必須明確列出會動到的「核心檔案清單」以及對現有 `GAME_RULES.md` 的影響。

## 2. 錯誤預防

- **編譯檢查**：完成程式碼寫入後，必須自動執行 `npm run build` 或對應測試腳本。
- **復原機制**：若修改導致 Build 失敗，Agent 應主動解釋原因並詢問是否需要復原（Rollback）。
