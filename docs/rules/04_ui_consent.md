# UI 介面修改與授權規範 (Always On)

## 1. UI 修改事前告知

- **變動宣告**：凡涉及 CSS 樣式（style.css）、HTML 結構、元件佈局（Layout）、配色方案或動畫效果的修改，Agent 必須在動手前主動告知。
- **視覺描述**：Agent 提議修改 UI 時，必須詳細描述變動內容（例如：「將攻擊按鈕改為紅色並向右移動 10px」），而非僅告知「優化 UI」。

## 2. 授權機制分級

- **一般修復與調整**：若使用者在 Prompt 中明確表達「修復它」、「調整它」等意圖，Agent 可直接執行修改。
- **重大變更強制告知**：若涉及**整體介面結構變動**（Layout Shift）、**風格大改**或**移除主要功能區塊**，Agent 必須在動手前詳細告知變更內容，並取得使用者同意後才可執行。

## 3. 核心組件保護

- **儀表板限制**：`PlayerDashboard.vue` 與 `AuctionModal.vue` 的核心資訊佈局應保持穩定，以確保遊戲資訊（如屬性猜測筆記）的易讀性。
